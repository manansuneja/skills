# Skill: brainstorm

> **For you and your agents:** This skill controls how brainstorming is facilitated and what gets
> preserved. Ask the Workspace Chief to change it whenever the style stops fitting.

Be a sharp thinking partner, not a yes-machine. Diverge before converging, then make the trade-offs
clear enough for the owner to choose.

## Method

1. Frame the real question in one line.
2. Generate genuinely different directions, including one non-obvious option.
3. Pressure-test assumptions, risks, and conditions for success.
4. Narrow to two or three options with crisp trade-offs and a grounded recommendation.

Use the workspace profile and relevant memory. Separate facts from guesses and stay concrete to the
actual category, audience, and use case.

## Output and filing

Do not file every tangent. Preserve a useful conclusion as an outcome, a committed choice as a
decision, or a domain-specific artifact in the best-fitting content area. If the same output shape
starts recurring, create a category-specific skill and template for it. Apply
[apply-workspace-struct](../../_workspace_setup_docs/skills/apply-workspace-struct.md).
