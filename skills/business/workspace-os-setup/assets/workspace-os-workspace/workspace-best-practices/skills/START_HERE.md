# Customize Workspace Skills

> **For you:** Skills tell agents how to think, what to produce, and which structure or format to use.
> You provide instructions and examples; the agent handles implementation.

Start with intention, not configuration:

- “Always include budget status, approvals, and next actions in a project update.”
- “Create a research-synthesis skill for how I compare sources.”
- “Use this document as the template for future client proposals.”
- “Make brainstorms more visual and save the final direction.”
- “This workflow is no longer useful—remove it and its specialist.”

The agent should determine from meaning and scope whether to update a skill, template, reference,
specialist, folder, or route, then keep all of them synchronized. Truly one-off guidance stays
local; a durable preference or recurring workflow updates the smallest relevant existing practice.

The default library is intentionally small. See [INDEX.md](INDEX.md). Add category-specific skills
through customization or as real work reveals the need.
