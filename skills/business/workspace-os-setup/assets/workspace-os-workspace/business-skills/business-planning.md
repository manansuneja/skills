# Skill: business-planning

Turn business context into a durable plan, release plan, sequencing recommendation, or now/next/later
view.

## Templates and references

Before using the default structure, check:

- [business-practices/templates/plan.md](../business-practices/templates/plan.md) for the current plan template.
- [business-practices/references/](../business-practices/references/INDEX.md) for Owner-provided plan, release plan, sequencing, or planning
  examples.

If the owner adds a better plan template or reference example, follow that and keep this skill's
sequencing rules.

## How to do it well

- Plans should connect strategy to execution: outcomes, offer briefs, decisions, brand and delivery, and data.
- Make sequencing explicit. Dependencies and confidence matter as much as dates.
- Use dates only when there is a real commitment or useful planning horizon.
- Link follow-on client updates when a plan should be communicated.

Write to `{{DOCS_ROOT}}/plans/<MMM-DD-YYYY>-<topic>.md`, update
[the plans index](../{{DOCS_ROOT}}/plans/INDEX.md), and apply
