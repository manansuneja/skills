# Skill: metrics

Turn metrics, research counts, experiment notes, dashboards, or quantitative user evidence into a
durable business insight.

## Templates and references

Before using the default structure, check:

- [business-practices/templates/metric-insight.md](../business-practices/templates/metric-insight.md) for the current metric insight template.
- [business-practices/references/](../business-practices/references/INDEX.md) for Owner-provided metric readouts, dashboard notes, experiment
  examples, or data storytelling preferences.

If the owner adds a better data template or reference example, follow that and keep this skill's
evidence and caveat rules.

## How to do it well

- Separate what the data shows from what you infer.
- State confidence and caveats. Do not invent data that is not present.
- Connect implications to outcomes, offer briefs, decisions, or plans when relevant.
- Preserve raw data under an artifact-local `raw/` folder when the source is a file or paste.

[the metrics index](../{{DOCS_ROOT}}/metrics/INDEX.md), and apply
