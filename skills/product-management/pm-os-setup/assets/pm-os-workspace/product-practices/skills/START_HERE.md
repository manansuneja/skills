# Start Here — Product Skills

> **For you:** Product skills teach PM OS how you want recurring product work handled. You do not
> need to edit them manually.

Tell the Chief PM what should change:

- “Make every PRD start with the customer problem and evidence.”
- “Use this example as the format for future meeting summaries.”
- “Create a skill for experiment readouts.”
- “Remove the roadmap skill if this workspace no longer uses it.”
- “Make updates shorter and more executive-friendly.”

The agent should update the skill, templates or references, affected specialist and Chief PM route,
and all relevant indexes. It should tell you what changed and offer one realistic test prompt.

## Minimal core

The starter library contains summarizing notes, brainstorming, synthesizing outcomes, writing PRDs,
and documenting product context. Personalization may remove an irrelevant core item or add focused
skills for the product stage and PM scope. Less is more; repeated usage should earn new machinery.

## Templates and references

- [templates/](../templates/INDEX.md) contains exact reusable structures.
- [references/](../references/INDEX.md) contains examples, source material, product principles,
  voice, and style guidance.

Add a file or describe your preference, then tell the agent what it should influence. It should
connect the material to the relevant skill automatically.

## What stays synchronized

When a skill changes, the agent checks:

- [the skill index](INDEX.md);
- [templates](../templates/INDEX.md) and [references](../references/INDEX.md);
- [Chief PM routing](../../agents/pm-chief.md);
- [the specialist index](../../agents/sub-agents/INDEX.md) and matching specialist;
- affected product areas and their indexes.

See [manage-workspace-skills.md](../../_workspace_setup_docs/skills/manage-workspace-skills.md) for
the protected maintenance playbook.
