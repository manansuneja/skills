# PM OS Setup

This folder is the onboarding helper. It is temporary. Once setup is done, you can delete `_setup/`.

## Begin

Tell your AI agent:

> Read `_setup/AGENTS.md` and interview me to personalize this PM OS.

This is like the onboarding step after creating a new dev project. If you already created/opened this
workspace folder, setup happens here. If you have not created a folder yet, ask the agent to create a
new PM OS workspace using the project name.

The goal is not to explain the folder structure. The goal is to give the workspace enough product
context that the Chief PM, sub-agents, product docs, and skills start behaving like they belong to
your actual product.

## What setup should capture

- What the product or project is.
- Who it is for.
- The problem it solves and why it matters.
- The current product stage, bet, or priority.
- How you like meetings, PRDs, decisions, and brainstorming handled.
- What existing docs or context should shape the workspace.
- Which tool you use day to day, only so the right lightweight adapter can be enabled.

After setup, start using the Chief PM. The workspace should get richer from real notes, PRDs,
decisions, and skill updates over time.
