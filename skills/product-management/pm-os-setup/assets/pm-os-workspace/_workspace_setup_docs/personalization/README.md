# PM OS Setup

> **For you:** Answer a lightweight product-context intake. Your agent handles structure, files,
> cleanup, and the final safe folder rename.

This folder is the onboarding helper. It is temporary. Your agent removes it automatically once setup
is done.

## Begin

Tell your AI agent:

> Customize my workspace.

This is like the onboarding step after creating a new dev project. If you already created/opened this
workspace folder, setup happens here. If you have not created a folder yet, ask the agent to create a
new PM OS workspace using the project name.

The goal is to give the workspace enough context that the product docs, product practices, Chief PM,
and useful specialists fit your product stage and responsibilities without unnecessary clutter.

## What setup should capture

- What the product or project is, who it is for, and what matters right now.
- The current stage, bet, goal, or active decision if you know it.
- Existing files, folders, pasted notes, docs, links, examples, decisions, or constraints to use.
- The kinds of PM work and tracking that matter now—and anything you do not need.
- Any format or style you already want followed.

After setup, start using the Chief PM. Ask it to add folders, trackers, skills, templates, or
specialists as the work evolves. If you add something manually, the next agent should integrate it
automatically.
