# skills/ - Index

Skills are repeatable judgment: how this PM wants a kind of work done. Agents apply them; the PM
refines them over time. As you do more reps, move taste from your head into these files. Skills are
the neutral *how*; the personas in [agents/sub-agents/](../agents/sub-agents/INDEX.md) are the *who*
that apply them.

| Skill | Use it to | Applied by |
|---|---|---|
| [apply-pmos-struct.md](apply-pmos-struct.md) | Name files, place folders, keep indexes current | Everyone (the glue) |
| [summarize-notes.md](summarize-notes.md) | Turn raw notes into a clean, structured summary | meeting-summarizer |
| [brainstorm.md](brainstorm.md) | Diverge then converge on options with trade-offs | brainstorm-partner |
| [to-prd.md](to-prd.md) | Turn notes/discussion into a structured PRD | prd-writer |
| [document-product-context.md](document-product-context.md) | Capture durable product knowledge and keep docs/indexes current | documentation-steward |
| [manage-workspace-skills.md](manage-workspace-skills.md) | Create or update workspace skills over time | skill-librarian |

**Adding a skill?** Drop a `lower-kebab-case.md` file here, add a row above, and reference it from
the agent(s) that should apply it.
